# Memory Capture Lite

Capture only lessons that are likely to help again.

## Good capture candidates

- repeated user preferences
- recurring validation steps
- stable project-specific conventions
- known pitfalls that caused rework

## Bad capture candidates

- one-off situational details
- transient debugging notes
- facts already obvious from the code

## Capture targets

Prefer one of:

- `AGENTS.md`
- a nearby skill reference
- a concise repo doc tied to repeated work

Keep captures short and actionable.
