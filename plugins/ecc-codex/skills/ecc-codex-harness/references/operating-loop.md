# ECC Codex Operating Loop

Use this loop for substantial work.

## 1. Frame

- What is the actual user outcome?
- What is the fastest safe path?
- What must be known before editing?

## 2. Discover

- inspect repo context
- inspect the current implementation
- inspect user-facing risks

## 3. Decide

- what to do locally now
- what, if anything, to delegate
- what validation will prove success

## 4. Execute

- make the smallest coherent change set
- keep edits aligned with repo conventions

## 5. Validate

- run the cheapest meaningful checks first
- escalate only if more confidence is needed

## 6. Capture

- if a lesson is reusable, record it concisely
- otherwise leave no extra process debris
